Сценарий 1: Добавление новой книги
Название теста: Проверка функциональности добавления книги в библиотеку

Цель тестирования: Убедиться, что библиотекарь может добавить новую книгу 
и что книга корректно сохраняется в системе.

Предпосылки:
- API сервер запущен и доступен
- Тестовые данные готовы

Шаги тестирования:
1. Отправить POST запрос на /api/books с данными новой книги
   Ожидаемый результат: Сервер возвращает статус 201 (Created)

2. Проверить тело ответа
   Ожидаемый результат: В ответе содержатся все поля книги: id, title, author, 
   isbn, year, quantity, available, created_at

3. Проверить, что все поля соответствуют отправленным
   Ожидаемый результат: title = "Новая книга", author = "Иван Иванов", 
   isbn = "978-5-12345-678-9", year = 2025, quantity = 5, available = 5

4. Проверить внутреннюю коллекцию books
   Ожидаемый результат: Книга с полученным id существует в коллекции books
