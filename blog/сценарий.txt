Сценарий 1: Создание нового поста
Название теста: Проверка функциональности создания нового поста

Цель тестирования: Убедиться, что авторизованный пользователь может успешно создать новый пост и что пост корректно сохраняется в системе.

Предпосылки:
- Пользователь зарегистрирован в системе
- API сервер запущен и доступен
- Тестовые данные готовы

Шаги тестирования:
1. Отправить POST запрос на /api/posts с данными нового поста
   Ожидаемый результат: Сервер возвращает статус 201 (Created)

2. Проверить тело ответа
   Ожидаемый результат: В ответе содержатся все поля нового поста: id, title, content, author_id, created_at, comments_count, likes

3. Проверить, что title и content соответствуют отправленным
   Ожидаемый результат: title = "Новый тестовый пост", content = "Содержание тестового поста"

4. Проверить внутреннюю коллекцию posts
   Ожидаемый результат: Пост с полученным id существует в коллекции posts

5. Проверить счетчик постов у автора
   Ожидаемый результат: У автора posts_count увеличился на 1

- Подготовлены некорректные данные для запросов

Шаги тестирования:
1. Отправить GET запрос на несуществующий пост
   Ожидаемый результат: Статус 404 (Not Found), сообщение "Post not found"

   Ожидаемый результат: Статус 400 (Bad Request), сообщение о required fields

   Ожидаемый результат: Статус 404 (Not Found), сообщение "Post not found"

4. Отправить PUT запрос на несуществующий комментарий
   Ожидаемый результат: Статус 404 (Not Found), сообщение "Comment not found"

5. Отправить DELETE запрос на несуществующий пост
   Ожидаемый результат: Статус 404 (Not Found), сообщение "Post not found"
